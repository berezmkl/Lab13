# Лабораторна робота №13 — Тестування продуктивності з Apache JMeter

## Мета
Ознайомлення з основами тестування продуктивності веб-додатків та практичне застосування інструменту Apache JMeter.

## Основні висновки

- Успішно розгорнуто робоче середовище Apache JMeter та налаштовано структуру проєкту.
- Створено три типи тестових планів: навантажувальний (Load), стресовий (Stress) та витривалості (Endurance).
- При навантажувальному тесті (10 користувачів) середній час відповіді сервера склав ~200 мс, відсоток помилок — 0%.
- При стресовому тесті (200 користувачів) середній час відповіді зріс до ~400 мс, що підтверджує деградацію продуктивності під високим навантаженням.
- Автоматизовано запуск тестів через GitHub Actions з генерацією HTML-звіту.
- Інструмент JMeter довів свою ефективність для виявлення меж стабільності вебсервера.

## Структура репозиторію

- `test-plans/` — JMX файли тестових планів
- `test-data/` — CSV файли з тестовими даними
- `results/` — результати виконання тестів (.jtl)
- `reports/` — HTML звіти
- `.github/workflows/` — конфігурація CI/CD
